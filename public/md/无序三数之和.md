### 一、题目

给你一个整数数组 `nums` ，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k`且 `j != k` ，同时还满足 `nums[i] + nums[j] + nums[k] == 0` 。请你返回所有和为 `0` 且不重复的三元组。

注意：答案中不可以包含重复的三元组。

示例：

```bash
示例 1：
输入：nums = [-1,0,1,2,-1,-4]
输出：[[-1,-1,2],[-1,0,1]]
解释：
nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0 。
nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0 。
nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0 。
不同的三元组是 [-1,0,1] 和 [-1,-1,2] 。
注意，输出的顺序和三元组的顺序并不重要。

示例 2：
输入：nums = [0,1,1]
输出：[]
解释：唯一可能的三元组和不为 0 。

示例 3：
输入：nums = [0,0,0]
输出：[[0,0,0]]
解释：唯一可能的三元组和为 0 。
```

### 二、解题思路

首先还是那句话，顺序不重要，那我们就规定一个顺序，而且因为这个题目不需要输出下标，所以可以直接sort排序

因为答案中不可以包含重复的三元组，所以在遍历的时候，如果当前值和下一个值相等，就跳过，不重复计入

我们可以把三数之和变成两数之和，把第一个元素`num1`摘出来遍历，剩下两个元素就变成了需要满足双数之和为`-num1`

### 三、正确答案

```js
var threeSum = function (nums) {
  const result = [];
  const n = nums.length;

  // 1. 排序（必须按数字排序）
  nums.sort((a, b) => a - b);

  // 只需要遍历到n-2，因为要预留3个元素
  for (let i = 0; i < n - 2; i++) {
    // 2. 剪枝优化
    // 如果当前数字大于0，则三数之和一定大于0，不可能有解，所以结束循环
    if (nums[i] > 0) break;

    // 3. 去重：跳过重复的nums[i]  这里的i>0是i-1的前提，无特定含义
    // if(nums[i]===nums[i+1])是前瞻性跳过，会漏掉需要重复元素作为第一个数的解。
    //正确的写法是回顾性跳过：先处理第一个出现的值跳过后续相同的值,既保证不重复，又保证不漏解
    if (i > 0 && nums[i] === nums[i - 1]) continue;

    // 4. 双指针查找
    let left = i + 1;
    let right = n - 1;
    const target = -nums[i]; // 需要找到两数之和为-nums[i]

    while (left < right) {
      const sum = nums[left] + nums[right];

      if (sum === target) {
        // 找到答案
        result.push([nums[i], nums[left], nums[right]]);

        // 去重：跳过重复的left和right(这里也是，去重操作放到找到答案之后)
        while (left < right && nums[left] === nums[left + 1]) left++;
        while (left < right && nums[right] === nums[right - 1]) right--;

        // 移动指针继续寻找其他解（这步别忘了）
        left++;
        right--;
      } else if (sum < target) {
        left++;
      } else {
        right--;
      }
    }
  }

  return result;
};
```
