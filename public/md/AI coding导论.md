### Agent 智能体

智能体（AI Agent）是指能够**感知和改变外部环境**、**进行自主推理、决策和执行任务**的智能软件系统。它以大语言模型（LLM）为“大脑”，通过调用外部工具（如搜索引擎、API、代码解释器）来主动完成复杂目标，具有自主性、适应性和持续学习的能力。

核心特征与能力：

- 自主性（Autonomy）： 不需要时刻人工干预，能独立制定计划并执行任务。
- 感知环境（Perception）： 能感知环境中的数据变化（如文本、代码、图像）。
- 推理与规划（Reasoning & Planning）： 将复杂任务分解为可执行的步骤。
- 记忆能力（Memory）： 能存储和回忆历史交互信息，不断优化行动。
- 工具使用（Tool Use）： 能够使用软件工具完成特定工作（如文件读写、网络搜索）。

智能体的核心架构：

- 一个完整的智能体通常由以下部分组成：
- 大脑（Brain）： 大语言模型，负责理解、推理和决策。
- 感知（Perception）： 接收外部输入信号。
- 记忆（Memory）： 存储历史工作流和知识。
- 行动（Action）： 使用工具执行动作。

应用场景与举例：

- 智能个人助手： 自动排程、阅读邮件、规划行程。
- 自动化办公： 自动编写代码、分析数据并生成报告。
- 智能客服： 理解用户意图并解决复杂问题。
- 自动化驾驶： 实时感知道路情况并采取行动。

与传统AI的区别：

传统AI通常是“输入-输出”模式（如回答一个问题），而智能体是一个持续工作的系统，能够为达成一个目标连续采取多步行动，甚至自主纠错。

### vibe coding

Vibe coding（氛围编程/感觉编程）是一种基于大语言模型（LLM）的全新软件开发实践，由OpenAI 前合伙人Andrej Karpathy于2025年初提出。它强调利用AI工具（如Cursor、Claude）通过自然语言描述需求，即“用感觉指挥AI生成代码”，无需深入理解具体语法，甚至“忘记代码存在”，追求快速开发和即兴创作。

这种方式降低了编程门槛，使非技术背景人员也能利用AI构建软件，但对于复杂系统开发需谨慎对待**代码安全与质量问题**。

核心含义与特征：

- 以“对话”取代“语法”： 开发者的核心工作转变为使用人类日常语言（如英语或中文）指导AI编写代码，核心精神是“以沟通替代语法”。
- 佛系写码，信任AI： 将代码的编写、调试和优化全部抛给AI，甚至可以直接接受AI的输出而不需要逐行检查。
- 适用场景： 非常适合原型设计（Prototyping）、一次性周末项目（Throwaway weekend project）或快速启动新功能。

争议与挑战：

- 代码质量风险：可能生成看似正确但存在隐藏问题的代码。
- 认知弱化：过度依赖可能导致开发者对底层逻辑生疏。
- 调试困难：当AI生成复杂代码时，理解与调试成本可能更高。

Vibe Coding反映了AI工具如何改变开发者与代码的交互方式，本质是将编程从“精确指令艺术”部分转向“意图沟通艺术”。它并未取代传统编程技能，但要求开发者掌握新的能力：<mark>清晰表达意图、批判性评估AI输出、高效迭代优化。</mark>

### MCP

一句话：agent和工具服务之间调用的约定

大语言模型 (LLM) 功能强大，但存在两大局限性：**其知识在训练时就已固定，且无法与外部世界交互**。这意味着它们无法访问实时数据，也无法执行如预订会议或更新客户记录等操作。

Model Context Protocol (MCP) 是一种旨在解决该问题的开放标准。MCP 由 Anthropic 于 2024 年 11 月推出，为 LLM 与外部数据、应用和服务之间的通信提供一种安全且标准化的“语言”。它充当桥梁，使 AI 不再局限于静态知识，而成为一个能够检索当前信息并执行操作的动态智能体，从而提升其准确性、实用性与自动化能力。

MCP 为 AI 应用创建了**标准化**的双向连接，使 LLM 能够轻松连接各种**数据源和工具**。MCP 建立在**工具使用和函数调用**等现有概念之上，并对其进行标准化。这样就无需为每个新的 AI 模型和外部系统创建自定义连接。它使 LLM 能够使用最新的真实数据、执行操作，并访问其原始训练中未包含的专业功能。

使用 MCP 的优势:

- 减少幻觉
- 提升 AI 的实用性与自动化能力
- 为 AI 提供更便捷的连接

### Agent Skills

一句话：prompt动态加载器

Agent Skills 是一种标准化的程序性知识封装格式。如果说 MCP 为智能体提供了"手"来操作工具，那么 Skills 就提供了"**操作手册**"或"SOP（标准作业程序）"，教导智能体如何正确使用这些工具。

Agent Skills主要用于提高AI执行特定任务的精确度，并且避免冗长的提示词和上下文爆炸的问题。

特点：格式化、可复用、按需/动态加载。

节省token：所有skill的名字和描述（元信息）始终对模型可见，但具体的指令内容（指令层）只有在skill被选中后，才会被加载出来给模型看

reference（资源层）：条件触发、按需加载、渐进式披露prompt（比如会议总结助手skill ，如果提到了钱，又会去读取集团财务手册，否则不会占用上下文）

script（资源层）：相当于独属于这个skill的mcp，可以执行轻量的脚本

> 注意一点，虽然reference和script都属于agent skills的高级功能，但是它们对于模型上下文的影响是截然不同的，reference是读，它会把内容加载到上下文里面，所以是会消耗token的，script是跑，它只会被执行，不会占用模型的上下文

### RAG

Retrieval-Augmented Generation 检索增强生成

是一种通过在生成回答前先从外部知识库检索相关信息，来增强大语言模型（LLM）能力的技术框架 。它解决了模型知识滞后、幻觉问题以及私有数据缺乏的问题，无需重新训练模型即可提供准确、实时且可追溯的回答。

知识库构建：分片->索引（①通过embedding模型将片段文本转换为向量②将文本片段和片段向量存入向量数据库中）

提问后（了解即可）：通过embedding模型将片段文本转换为向量->向量数据库->召回->重排->大模型产出最终答案
